# 《混沌工坊》1.1v 总结说明

本文档用于总结本轮 `1.1v` 整理工作的实际成果、反复遇到的问题，以及已经沉淀到 agent 约定中的经验。

## 1. 本轮整理目标

这次 `1.1v` 的核心目标不是单点加新功能，而是把《混沌工坊》现有原型整理成一套可以继续稳定迭代的基线：

- 统一项目目录与 Unity 工程入口。
- 统一设计文档、JSON 数据和 Unity 脚本说明。
- 把敌人、UI 接口、Demo 场景与生成资源补成可交接的原型骨架。
- 把今天反复踩到的流程问题写进 agent，减少后续重复返工。

## 3. 频繁遇到的问题

### 3.1 目录表述和实际工程不一致

今天最容易引发误判的问题，是旧路径写法和当前工程真实路径不一致：

- 旧说法：`Assets/Scripts/ChaosWorkshop`
- 现状：`Chaos Werkswinkel/Assets/ChaosWorkshop`

如果不先确认真实路径，很容易把“目录迁移”误认为“脚本丢失”。

### 3.2 根目录文档和 Unity 镜像文档容易漂移

当前仓库同时保留：

- 根目录主文档
- `Chaos Werkswinkel/Assets/` 下的镜像文档

如果只更新一侧，内容就会很快失同步，后续维护成本会越来越高。

### 3.3 中文文件在终端里容易乱码

今天在 PowerShell 环境里多次遇到：

- 中文文件名显示乱码
- Git 输出中文路径不直观
- 直接读取文档时编码不稳定

这类问题本身不难修，但会反复打断判断和整理节奏。

### 3.4 工具链可用性不稳定

今天反复遇到两个工具问题：

- `git` 不在默认 `PATH` 中，需要显式调用 Visual Studio 自带 Git。
- `rg` 在当前环境里不可用或被拒绝访问，需要退回 PowerShell 的 `Get-ChildItem` 与 `Select-String`。

这意味着后续 agent 不能默认依赖常用 CLI 工具都能直接工作。

### 3.5 Unity 生成文件很容易制造提交噪音

在整理和上传仓库时，最容易混入的是：

- `Library`
- `Temp`
- `Logs`
- `obj`
- `UserSettings`
- `.vs`
- 自动生成的 `*.csproj` 与 `*.sln`

这些内容会让提交记录失焦，也会掩盖真正有价值的脚本和资源变更。

## 4. 已沉淀到 Agent 的经验

本轮已把以下经验更新到 `AGENTS.md`：

- 仓库真实结构以 `Chaos Werkswinkel/` 为准，旧 `Assets/Scripts/ChaosWorkshop` 视为历史路径。
- 根目录文档与 `Chaos Werkswinkel/Assets/` 镜像文档必须同步维护。
- 在当前终端环境下处理中文文档时，优先使用 UTF-8 输出与 UTF-8 读取。
- 若 `git` 不在 `PATH` 中，优先使用 Visual Studio 自带 Git 路径。
- 若 `rg` 不可用，改用 PowerShell 文件遍历与文本检索。
- 上传仓库时应聚焦 `Assets`、`Packages`、`ProjectSettings` 等有效工程内容，并严格排除 Unity 生成目录与 IDE 噪音文件。
- 如果 Git 同时显示旧脚本路径删除和新工程路径新增，应先判断是否为结构迁移，而不是立即当成异常。

## 5. 当前 1.1v 的边界

`1.1v` 已经把原型整理到“可继续扩展、可交接、可进入下一轮开发”的状态，但还不是完整成品。当前仍明确属于后续项的内容包括：

- 海洛安完整的告解 / 赎罪 / 弃牌收益闭环。
- 完整混沌浪潮效果池的数据驱动接入。
- 多敌人正式遭遇生成与更完整的 HUD 提示。
- 更完整的剧情、奖励和弃牌选择 UI。

## 6. 后续建议

建议后续优先级如下：

1. 先补齐海洛安完整机制闭环，因为她最能体现慢速角色的设计方向。
2. 再把混沌浪潮效果池改成正式数据驱动，减少脚本和文档漂移。
3. 将多敌人战斗与敌人意图显示并入固定 Demo 验收流程。
4. 持续保持“根目录主文档 + Unity 镜像文档 + JSON 草案 + 原型脚本”四者同步。
