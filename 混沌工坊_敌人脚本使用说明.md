# 《混沌工坊》敌人脚本使用说明 1.1v

本文档说明当前 Unity 原型中的敌人数据与 AI 接法，基于 `Chaos Werkswinkel/Assets/ChaosWorkshop` 下的实际脚本整理。文件名保留 `v1`，正文版本统一为 `1.1v`。

## 1. 相关脚本

当前敌人系统主要由以下脚本组成：

- `Assets/ChaosWorkshop/Data/EnemyDefinition.cs`
  - 敌人 ScriptableObject 风格定义。
  - 包含 `EnemyTier`、`EnemyActionDefinition` 与 `EnemyActionKind`。

- `Assets/ChaosWorkshop/Data/EnemyLibrary.cs`
  - 代码版敌人数据库。
  - 负责按 `enemyId` 生成敌人数值与动作原型。

- `Assets/ChaosWorkshop/Battle/EnemyAI.cs`
  - 按 `enemyId` 决定具体行动逻辑。
  - 管理敌人临时状态，如标记、力量、浪潮后状态等。

同时会联动：

- `Assets/ChaosWorkshop/Battle/BattleManager.cs`
- `Assets/ChaosWorkshop/Battle/CombatUnit.cs`
- `Assets/ChaosWorkshop/Data/BalancePrototypeFactory.cs`

## 2. 当前支持的 `enemyId`

普通敌人：

- `rogue_puppet`
- `rustblade_hunter`
- `steam_guard`
- `gun_wanderer`
- `chaos_apprentice`

精英敌人：

- `broken_axle_knight`
- `redline_sniper`
- `furnace_steward`

Boss：

- `first_chaos_core`

## 3. 当前 AI 结构

`EnemyAI.TakeAction()` 会根据 `enemyId` 进入不同方法：

- `ActRoguePuppet()`
- `ActRustbladeHunter()`
- `ActSteamGuard()`
- `ActGunWanderer()`
- `ActChaosApprentice()`
- `ActBrokenAxleKnight()`
- `ActRedlineSniper()`
- `ActFurnaceSteward()`
- `ActFirstChaosCore()`

AI 完成行动后，会调用：

```csharp
battleManager.CompleteEnemyAction();
```

说明：

- 当前仍属于“半数据驱动”状态。
- `EnemyLibrary` 已提供敌人动作定义结构，但最终行为仍主要写在 `EnemyAI` 中。

## 4. 运行时状态字段

`EnemyAI` 当前维护的主要运行时字段：

- `powerStacks`
- `markedPlayer`
- `panicShot`
- `postWaveAction`
- `missedByDistanceCount`
- `lastUsedActionId`
- `repeatedTimelineControlCount`
- `randomDistanceActionsInRow`
- `lowHealthGuardUsed`
- `bossPhase`

这些字段分别服务于：

- 力量型敌人的蓄势
- 狙击类敌人的标记
- 远程敌人的惊惶后撤状态
- 混沌浪潮后的强化行动
- Boss 阶段切换与随机距离控制

## 5. 行为概览

### 5.1 失控傀儡

- 距离大于 2 时向玩家靠近。
- 距离小于等于 2 时攻击。
- 连续因距离无法命中后，下次命中伤害会变高。

用途：

- 最简单的教学近战模板。

### 5.2 锈刃追猎者

- 贴身后进行 2 段攻击。
- 不在近战距离时快速靠近。
- 会根据距离选择更积极的追击速度。

用途：

- 制造时间压力。

### 5.3 蒸汽盾卫

- 不在攻击范围内时加盾推进。
- 在攻击范围内时进行重击。
- 浪潮后首个动作会更偏向护盾收益。

用途：

- 教学“慢速敌人不等于弱，只是把威胁集中在关键回合”。

### 5.4 枪械游民

- 距离 4 到 9 时射击。
- 被贴近后会后撤，并强化下一次射击。
- 太远时会向前找射线。

用途：

- 教学远程敌与距离管理。

### 5.5 混沌学徒

- 会在玩家即将出手时尝试延后玩家行动条。
- 中近距离会直接攻击。
- 浪潮后可推进自身行动条。

用途：

- 教学行动条与浪潮先后手的价值。

### 5.6 断轴骑士

- 可通过被强制位移积累力量。
- 力量足够时会转化成强突进。
- 低血量时有一次护盾自保。

用途：

- 让击退不再是完全无代价的万能解。

### 5.7 红线狙击手

- 远距离先标记，再打出高伤狙击。
- 中距离改用腰射。
- 贴身时后撤并获得闪避。

用途：

- 强迫玩家主动压缩距离。

### 5.8 熔炉执事

- 每次浪潮自动获得大额护盾。
- 推进时也会叠小护盾。
- 护盾足够高时会把盾量转成更高伤害。

用途：

- 强调浪潮不是只对玩家有利。

### 5.9 初代混沌炉心

- 依据生命比例切换三阶段。
- 阶段 1 偏稳定推进与护盾。
- 阶段 2 提速并强化拉近。
- 阶段 3 通过随机改距和行动条推进制造终盘混乱。

用途：

- 把第一章所有节奏课题整合到一场 Boss 战中。

## 6. 浪潮联动

`EnemyAI.OnChaosWaveResolved()` 当前会在浪潮后处理敌人专属逻辑：

- 所有敌人获得 `postWaveAction = true`
- 熔炉执事：直接获得 14 护盾
- 初代混沌炉心：
  - 阶段 1 获得 10 护盾
  - 阶段 3 随机改距并推进自身行动条

这让慢速敌人与 Boss 真正参与到 12 秒节奏中，而不是只让玩家受益。

## 7. 被位移后的处理

`EnemyAI.RegisterForcedMove()` 当前主要服务于断轴骑士：

- 当它被 `CardEffectType.Move` 强制位移时，会积累力量。
- 这与设计文档中的“击退有代价”方向保持一致。

说明：

- 目前这是通过通用 `Move` 效果触发。
- 如果后续增加专门的 `knockback` 字段，建议把所有强制位移统一收口到一个接口里。

## 8. 场景里如何使用

### 手动挂载

1. 创建敌人物体。
2. 挂 `CombatUnit`。
3. 挂 `EnemyAI`。
4. 设定 `EnemyAI.enemyId`。
5. 启动前执行：

```csharp
enemyAI.Configure("gun_wanderer");
enemy.InitializeRuntime(14f);
```

6. 再调用：

```csharp
battleManager.StartBattle(player, enemy, deck);
```

### 用工厂生成

更推荐原型阶段直接调用：

```csharp
GameObject enemyObject = new GameObject("Gun Wanderer");
CombatUnit enemy = BalancePrototypeFactory.CreateRuntimeUnit(enemyObject, "gun_wanderer", Team.Enemy);
enemy.InitializeRuntime(14f);
```

工厂会自动：

- 设置敌人基础面板
- 配置阵营
- 挂载并初始化 `EnemyAI`

## 9. 当前边界

当前敌人系统的边界主要是：

- 行为虽有定义结构，但还不是完全数据驱动。
- 多敌人遭遇完整调度尚未完成。
- 敌人意图预览 UI 尚未正式接入。
- 部分设计文档中的机制，在脚本里是“方向已经建立”，但还未全部做满。

目前最值得继续推进的方向：

1. 把 `EnemyActionDefinition` 真正用于驱动行为，而不只是做定义容器。
2. 为敌人加入意图显示文本或图标。
3. 完善多敌人目标选择、排序和遭遇生成。
4. 让浪潮效果与敌人反应进一步数据化。
